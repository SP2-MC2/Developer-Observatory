Developer Observatory - Documentation

Preconditions:

  • Python 3
  • Flask

We created a tool to help you generate the task files. A notebook file always has to follow the scheme:

  • Introduction (1 markdown cell)
  • any number of tasks (1 markdown cell + 1 code cell)

If you want to change this, you'll have to manually modify the JavaScript in instance_server/template/custom.js

If you are using a completly different set of tasks, for example, symetric vs asymetric encryption, then make sure that both files contain the same amount of cells+tasks. Incase that you have a different amount of tasks, you can also modify instance_server/template/custom.js to return the correct amount depending on the amount of cells in the document ($('.cell').length)


1) Start the notebook generator

Install the required dependencies with pip3 install -r requirements.txt, run python3 notebookgen.py and open http://localhost:5000 in your webbrowser.

2) Generate Cells

Switch to the Cells tab and click on "New Cell".

(1) The building blocks of Jupyter notebooks are Cells. Click on New Cell under Cells to create your first one.
You may now create Markdown (text only) cells and Code cells. A task will always need a Markdown cell followed by a Code cell except for the introduction which consists of a single Markdown cell.

(2) We create a cell to introduce our study. We use placeholders enclosed in percentage signs. %website% and %URL% are going to be replaced depending on the condition. %task1.description% dynamically references the second task since the first task is going to be this introduction and the tasks after that are going to change order.

(3) Here we can see all cells we created so far. Next we create cells for our first programming task.

(5) This is our first code cell, the heart of our programming task. Participants will be able to edit this code and execute it. Mandatory: Give the Code cells a metadata field called "tasknum" with a number so that the task can be identify by the software.

3) Generate Tasks

Switch to the Tasks tab and click on "New Task" and select the appropriate cells for the task.

(6) Now that we created some cells, we have to incorporate them into tasks.

(7) Our introduction task only contains the single cell with our introduction markdown.

(8) The first programming task consists of the two cells we created last.

(9) You might want to come up with some more cells and tasks.

4) Create Conditions

Switch to the Conditions tab and create conditions which are used to replace placeholders in cells.

(10) Time to create some conditions for our study.

(11) The official python documentation is our first condition. The website and URL values are going to replace the %website% and %url% placeholders we used earlier in our introduction cell. Placeholders can also be used in code cells to e.g. import different libraries depending on condition.

(12) Our second condition is Stack Overflow. Keep in mind this is just an example and do not think to much about the merit of this study.

5) Generate Notebooks

The last step is to generate the notebooks. Switch to the Notebooks tab and create a new notebook template. You can now select which tasks should be included in this notebook file. You can also select if certain tasks should be at a fixed position. The intro should always be fixed to the first position, followed by tasks in latin squares order.

(13) The actual notebook files are created from notebook templates.

(14) We set rules for the task order. We want our introduction to always come first, but the order of the tasks after that should be varied according to the Latin Squares method.

(15) Click on Generate to create notebook files from the template.

(16) Here we can select the conditions we want to use. Clicking on refresh shows us which files will be generated from the options we selected before we actually generate the files. The task order in each notebook is shown in brackets in the filename.

6) Jupyter

After generating the notebook file, it can be used with Jupyter.
